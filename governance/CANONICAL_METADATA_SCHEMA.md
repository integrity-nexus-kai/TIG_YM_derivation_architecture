# Purpose

This document defines the canonical metadata structure for all TIG documents.

Its purpose is to preserve consistency, dependency clarity, governance compatibility, and structural readability across the complete TIG research architecture.

---

# Core Principle

Every canonical TIG document must begin with a standardized metadata block.

Metadata must remain:

- minimal,
- explicit,
- dependency-compatible,
- and structurally interpretable.

---

# Canonical Metadata Order

All metadata fields must appear in the following order:

1. Title
2. Author
3. Layer
4. Status
5. Last Updated

No reordering is allowed.

---

# Mandatory Metadata Fields

The following fields are mandatory for all canonical TIG documents:

## Metadata

Title: ...
Author: Kai Stefan Dietrich
Layer: ...
Status: ...
Last Updated: YYYY-MM-DD

---

# Canonical Layer Names

Allowed layer values:

- Definitions
- Derivation Chain
- Horizon Structure
- Admissibility
- Operator Theory
- Governance
- Shared

No additional layer names may be introduced without governance review.

---

# Canonical Status Values

Allowed status values:

- primitive definition
- structural hypothesis
- theorem candidate
- partial derivation
- canonical reference
- rigorous result
- external speculation

No unofficial status values may be introduced.

---

# Date Format Rule

Dates must use ISO format:

YYYY-MM-DD

Example:

2026-05-10

---

# MRP Classification Block

Documents beyond primitive definitions should include an MRP classification block.

Recommended structure:

# MRP Classification

MRP Layer:
...

Dependency Level:
...

Rigorous Status:
...

Allowed Scope:
...

Forbidden Expansion:
...

---

# Optional Metadata Extensions

The following fields are optional:

- Dependency Notes
- Canonical References
- Revision Notes

Optional fields must appear only after the mandatory metadata block.

---

# Metadata Minimality Principle

Metadata must never:

- imply theorematic validity,
- introduce unsupported terminology,
- or replace formal derivation.

Metadata exists only for structural governance.

---

# Metadata Consistency Rule

Metadata terminology must remain consistent with:

- canonical terminology registry,
- dependency map,
- operator principles,
- and claim boundary protocol.

---

# Legacy Document Rule

Older documents created before governance consolidation may temporarily violate the canonical metadata schema.

Such documents must later undergo controlled normalization.

Normalization must preserve:

- original structural intent,
- dependency consistency,
- and terminology stability.

---

# Canonical Governance Rule

No document becomes canonical unless its metadata structure is governance-compatible.

---

# Current Status

The current TIG architecture is undergoing metadata normalization and governance consolidation.

This process remains ongoing.
