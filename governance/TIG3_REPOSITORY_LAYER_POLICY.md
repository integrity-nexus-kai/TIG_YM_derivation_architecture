# TIG3 Repository Layer Policy

**Repository:** `TIG_YM_derivation_architecture`
**Path:** `governance/TIG3_REPOSITORY_LAYER_POLICY.md`
**Status:** Active repository governance policy
**Classification:** Canonical governance layer

---

# 1. Purpose

This document defines the repository-layer separation policy for the TIG3 operator-theoretic research architecture.

The purpose is to distinguish between:

- canonical public-facing structures,
- and internal exploratory research structures.

This separation is intended to preserve:

- mathematical clarity,
- publication discipline,
- exploratory freedom,
- and long-term operator-theoretic consistency.

---

# 2. Repository Layer Structure

The repository is divided into two primary layers.

---

## 2.1 Canonical Public Layer

Purpose:

```text
stable public-facing mathematical architecture
```

Characteristics:

- publication-oriented,
- terminology-stabilized,
- dependency-controlled,
- claim-audited,
- mathematically conservative.

Typical directories:

```text
paper/
core/
governance/
overview/
```

Only stabilized and internally audited structures should appear here.

---

## 2.2 Internal Research Layer

Purpose:

```text
active exploratory stress-testing environment
```

Characteristics:

- exploratory,
- failure-oriented,
- mathematically experimental,
- noncanonical,
- internally iterative.

Typical directories:

```text
internal/operators/
internal/failure_analysis/
internal/gauge_stress_tests/
internal/spectral_risks/
```

This layer may contain:

- unresolved operator programs,
- speculative compatibility pathways,
- stress-test documents,
- instability analyses,
- failed constructions,
- incomplete mathematical routes.

---

# 3. Canonical Inclusion Rule

A document should enter the public canonical layer only if:

1. terminology is stabilized,
2. dependency structure is explicit,
3. claim classification is audited,
4. speculative language is minimized,
5. unresolved assumptions are identified,
6. mathematical positioning is defensible.

---

# 4. Internal Research Rule

Internal documents are allowed to:

- explore unstable pathways,
- investigate possible failures,
- test incomplete structures,
- analyze incompatibilities,
- evaluate speculative operator mechanisms.

However:

```text
internal exploratory status must remain explicit
```

at all times.

---

# 5. Current Layer Assignment

## Canonical Public Layer

Current public canonical structures include:

- D1--D12,
- canonical governance files,
- operator-core overviews,
- dependency structures,
- notation stabilization,
- claim-classification audits.

---

## Internal Research Layer

Current internal-research structures include:

- D13 and later stress-test files,
- closability analyses,
- semiboundedness analyses,
- essential-spectrum analyses,
- gauge-compatibility analyses,
- gauge-covariant domain programs.

These structures remain exploratory.

---

# 6. Migration Policy

Documents may move from:

```text
internal
→ canonical
```

only after:

- mathematical stabilization,
- internal consistency auditing,
- terminology hardening,
- dependency verification,
- and claim-classification review.

Reverse migration is also permitted if instability is discovered.

---

# 7. Scientific Integrity Principle

The purpose of the dual-layer architecture is not secrecy.

Its purpose is:

```text
controlled mathematical development
```

without confusing exploratory structures with stabilized results.

---

# 8. Current Repository Position

The TIG3 repository should currently be interpreted as:

```text
an exploratory operator-theoretic research program
with a stabilized canonical presentation layer
and a separate internal stress-testing layer
```

rather than as a finalized mathematical theory.

---

# 9. Conclusion

The repository-layer policy establishes:

```text
canonical stability
+
internal exploratory freedom
```

inside a unified operator-theoretic research architecture.
