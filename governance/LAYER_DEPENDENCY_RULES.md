# Layer Dependency Rules

## Principle

All theorematic development must proceed strictly upward through the canonical layer hierarchy.

Reverse dependency usage is forbidden.

---

# Canonical Layer Hierarchy

Layer 1:
definitions/

Layer 2:
derivation_chain/

Layer 3:
horizon_structure/

Layer 4:
admissibility/

Layer 5:
operator_theory/

Layer 6:
ym/

---

# Dependency Rule

A layer may depend only on:
- itself,
- or lower layers.

Dependencies on higher layers are forbidden.

---

# Forbidden Structures

Forbidden:
- circular theorematic justification
- retroactive admissibility injection
- YM-based TIG validation
- undefined operator assumptions
- speculative cross-layer transfer

---

# Recovery Procedure

If cross-layer contamination occurs:

1. stop theorematic expansion,
2. identify illegal dependency,
3. remove unsupported assumptions,
4. return to last admissible layer,
5. reconstruct upward minimally.

---

# Principle of Minimal Closure

No layer may expand before the lower layer stabilizes.
