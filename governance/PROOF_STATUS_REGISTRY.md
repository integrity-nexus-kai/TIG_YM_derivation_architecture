# PROOF STATUS REGISTRY

## Purpose

This registry defines the canonical proof-status classification system for the TIG, TIG3-Vacuum, and YM derivation programs.

Its purpose is:

- to separate proven results from conjectural structures,
- to prevent premature theorem claims,
- to provide repository-wide status transparency,
- to support external auditing,
- to maintain scientific traceability.

---

# Status Classes

## PROVEN

Definition:

A statement is classified as PROVEN if:

- a complete derivation exists,
- all assumptions are explicitly documented,
- no unresolved proof dependencies remain,
- independent audit review identifies no open gaps.

Status Color:

🟢 Green

---

## PARTIALLY PROVEN

Definition:

A statement is classified as PARTIALLY PROVEN if:

- substantial derivation exists,
- core components are verified,
- one or more proof dependencies remain unresolved.

Status Color:

🟡 Yellow

---

## DERIVED

Definition:

A statement is classified as DERIVED if:

- it follows from documented assumptions,
- the derivation chain is complete,
- rigorous proof validation has not yet been completed.

Status Color:

🟡 Yellow

---

## NUMERICALLY VERIFIED

Definition:

A statement is classified as NUMERICALLY VERIFIED if:

- computational evidence exists,
- simulations support the claim,
- no rigorous proof currently exists.

Status Color:

🟠 Orange

---

## CONJECTURAL

Definition:

A statement is classified as CONJECTURAL if:

- it is supported by heuristic reasoning,
- no complete derivation currently exists,
- proof obligations remain open.

Status Color:

🔴 Red

---

## OPEN

Definition:

A statement is classified as OPEN if:

- no proof currently exists,
- no accepted derivation currently exists,
- the problem remains unresolved.

Status Color:

⚫ Black

# Audit Rule

Any document claiming:

- proof,
- theorem,
- exact solution,
- solved problem,

must explicitly reference its current registry status.

Documents lacking status classification are considered non-canonical until reviewed.

---

# Governance Requirement

All future research outputs must be assigned one of the following statuses:

- PROVEN
- PARTIALLY PROVEN
- DERIVED
- NUMERICALLY VERIFIED
- CONJECTURAL
- OPEN

before inclusion into canonical repository structures.
